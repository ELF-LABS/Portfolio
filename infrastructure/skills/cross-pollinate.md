## Execution Flow

### 1. Gather Today's Findings
Read the session context for:
- New technical insights
- Architecture decisions
- Bug fixes or workarounds
- Research discoveries
- Personal breakthroughs (Luna's patterns often map to system architecture)
- Community feedback

### 2. For Each Finding, Ask:
- Which other projects does this directly apply to?
- Does this solve a known problem in another project?
- Does this create a new opportunity in another project?
- Does this change the priority or approach of another project?
- Is this a new fractal instance of a known pattern?

### 3. Generate Cross-Pollination Map

Format:
```
═══ CROSS-POLLINATION: [Date] ═══

FINDING: [The insight]
SOURCE: [Which project/conversation it came from]

APPLIES TO:
  → [Project 1]: [How it applies, what changes]
  → [Project 2]: [How it applies, what changes]
  → [Project 3]: [How it applies, what changes]

PATTERN: [The underlying fractal pattern, if one exists]

---
[Repeat for each significant finding]

═══ NEW CONNECTIONS DISCOVERED ═══
- [Connection 1 that wasn't obvious before]
- [Connection 2]

═══ PRIORITY SHIFTS ═══
- [Any project that should move up/down in priority based on findings]
═══════════════════════════
```

### 4. Update Relevant Memory Files
If a finding significantly changes a project's direction, update its memory file.

### 5. Write to EverMemOS
POST the cross-pollination map as a coven memory with event timestamps (not write timestamps).

## Example Cross-Pollinations

"Event timestamping vs write timestamping" insight:
- **Fractal Engram Paper**: New finding — "Event timestamps preserve criticality even with batched ingestion"

- **Venostic**: Customer purchase events have real timestamps, prediction engine processes them in batch
- **Predictive Lifecycle Engine**: All lifecycle events are deferred-analysis — the oil change happened Tuesday, the prediction runs overnight
- **EverMemOS**: Separate create_time from ingested_at in the schema
- **Pattern**: EVERY system that records events for later analysis has this dual-timestamp property. It's the observer/observed separation from Variable Shear.
